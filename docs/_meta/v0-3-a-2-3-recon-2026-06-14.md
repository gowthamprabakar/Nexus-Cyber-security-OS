# v0.3 / Phase D — A-2.3 recon: CVE→runtime-process reachability (2026-06-14)

> **Status:** Recon findings (doc-only, NO code). The operator chose the **deep correlator**
> (Fork A → A2), the genuine +5.9pp earner. Per the directive, A-2.3 implementation does NOT
> start until the operator decides the 3 forks below (mirror the bounded-drain / A-2 pattern).

## 1. Decisive finding — the join is a single osquery SQL query

The package→loaded-library→PID join rides the **existing generic osquery wrapper** (D.3's
`runtime_threat/tools/osquery.py` — a verbatim `osqueryi --json` SQL passthrough, no table
allowlist). The correlator is **a SQL query + a result normalizer**, NOT new instrumentation:

```
deb_packages / rpm_packages / apk_packages  (.path of installed files)
   ↔ process_memory_map.path                 (mmap'd shared libs per PID)
   ↔ processes.pid / processes.path          (the running process)
```

**No new engine, no eBPF, no charter/shared change, no ADR.** D.1 has no process access today
(its Trivy scans are package _inventories_, not running processes); D.1 doesn't yet depend on
`nexus-runtime`.

## 2. Honest coverage gaps (these bound the +5.9pp — must be surfaced, never faked)

- **Distroless / no package DB:** no `deb/apk_packages` content → package side of the join is empty → reachability `indeterminate`, not "not reachable."
- **Static-linked Go/Rust:** no shared-lib mmaps → `process_memory_map` can't see the vulnerable lib (compiled in). Biggest false-negative class for cloud-native.
- **musl vs glibc / path divergence:** join key is a raw path → per-distro path normalization needed or matches silently miss.
- **Containers / PID+mount namespaces:** host osquery sees host packages, not the container's package DB → container reachability needs per-container/namespace-aware osquery (deepest gap).
- **Interpreted langs (Py/Node/Java):** `process_memory_map` shows the interpreter's libs, not the PyPI/npm package → falls back to `process_open_files` (weaker).

Net: the join is **real + high-value for dynamically-linked OS packages on glibc hosts**; degrades to `indeterminate` elsewhere. The +5.9pp is earned on the host-OS-package slice; other slices must emit an explicit `indeterminate` state.

## 3. Three forks for operator decision (BEFORE implementation)

### Fork A-2.3-1 — osquery wrapper placement (the bounded-drain-style decision)

- **(1a) Hoist `osquery.py` → `nexus-runtime` (RECOMMENDED):** shared, D.3 + D.1 both consume (osquery is now a 2-consumer primitive; mirrors `bounded_drain`/`llm_invariants`). The reachability SQL + correlator + normalizer stay D.1-local. Cost: a cross-package rewire of D.3's import (small blast radius). **No ADR** (bounded_drain hoisted without one; the wrapper already conforms to ADR-005).
- **(1b) Duplicate a thin wrapper in D.1:** zero D.3 blast radius, pure additive self-merge; but two copies drift (against the hoist instinct).
- **(1c) Sibling-read D.3 findings: REJECTED** — D.3 emits threat-filtered OCSF-2004 findings, not a package-linkable process inventory (= the shallow A1 that doesn't earn +5.9pp).

### Fork A-2.3-2 — reachability state model

- **(2a) Tri-state `reachable` / `not_loaded` / `indeterminate` (RECOMMENDED):** surfaces the §2 gaps honestly; +5.9pp earned on confirmed-reachable; indeterminate is labeled, never a false "not reachable."
- **(2b) Binary reachable/not-reachable:** simpler but dishonest for distroless/static/container/interpreted → operators ignore real CVEs. Not recommended.

### Fork A-2.3-3 — container/namespace scope for v0.3

- **(3a) Host-level osquery only (RECOMMENDED for v0.3):** earns reachability on EC2/VM hosts (the slice that genuinely works); in-container package DBs explicitly out-of-scope.
- **(3b) Per-container/namespace-aware osquery:** much larger, lower confidence → defer to a follow-up.

## 4. Schema (additive; substrate CLEAR)

A single `location` string is **insufficient** — reachability is a _list of processes per CVE_ + a per-result state. Add an **optional `reachability` field** to `AffectedPackage` (defaults None):
`{ state: "reachable"|"not_loaded"|"indeterminate", processes: [{pid, process_path, loaded_path}], evidence_query: str }`. Additive, non-breaking (`FINDING_ID_RE` untouched; OCSF 2002 dict gains keys only when present; offline fixtures byte-identical). **No charter/shared edit (#19/#29 CLEAR).**

## 5. Licenses (#23) + substrate dependency (#37)

- **osquery: Apache-2.0** — clean (#23 CLEAR). Subprocess model, same as the already-shipped Trivy. **No new Python dep** (wrapper is pure stdlib; option 1a adds only the internal `nexus-runtime` workspace dep).
- **Avoid eBPF** for reachability: bcc/BPF programs carry a **GPL-2.0** surface (#23 risk) + a kernel-version substrate dependency, for marginal gain over `process_memory_map`.
- **#37 (system dependency): YES — flag honestly.** Requires the `osqueryi` binary at runtime on the scanned host — the **same operator-provisioned-binary model already accepted for Trivy** (`shutil.which`). Wrapper degrades gracefully (binary-not-found → clean skip, not crash). Live lane `NEXUS_LIVE_*`-gated; CI uses a faked osquery result (D.3 precedent).
