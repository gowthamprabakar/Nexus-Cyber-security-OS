Why IAM Roles?

Obsidian utilizes the AssumeRole pattern to minimize risk. By using cross-account roles, you never share permanent secret keys with our platform, and access can be revoked instantly from your AWS Console.

bolt Autonomous Agents

To provide sub-second threat response, agents require permissions to modify Security Groups and IAM policies. This allows Obsidian to isolate a compromised S3 bucket or EC2 instance before data exfiltration occurs.
